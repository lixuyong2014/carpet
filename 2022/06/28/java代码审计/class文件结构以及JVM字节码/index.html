class文件结构

先写一个接口，命名为StudyClassInterface.java，里面就一个执行命令的方法

再写一个java文件，命名为StudyClass.java

为了更好的理解class文件结构，分别定义了各种成员变量以及对应的getter/setter方法还有一个打印变量信息的printField方法和一个执行系统命令的execCmd方法，还有一个主函数，执行系统命令然后返回结果

将StudyClassInterface.java和StudyClass.java放在同一目录，使用编译命令javac -g StudyClass.java编译成class文件，-g指定在生成的class文件中包含所有调试信息（行号、变量、源文件）

命令行

使用javap -c -l -v StudyClass.class 命令查看反编译的class文件

  • -c：进行反编译
  • -l：输出行号和本地变量表
  • -v：更详细的信息

反编译显示出来很多内容，大部分都是指令，这里显示的内容和class文件是有点不同的，主要是方便我们查看

前四行：

  1. 指定反编译文件
  2. 上次修改时间和文件大小
  3. MD5校验和
  4. 从哪个文件编译而来

第五行到第九行：

  1. 哪个类，实现的哪个接口
  2. 次版本
  3. 主版本
  4. 访问权限修饰符
  5. 常量池（重要）

常量池完了之后，进入方法区

1.

第一个方法是构造函数，然后构造函数的第二行是描述符，描述了函数参数和返回值，第三行依旧是访问权限修饰符，第四行是代码区，代码区又包指令和行号表，以及本地变量表

其他方法同样是这个结构，有些方法可能会多一个异常表，比如execCmd方法里有个异常表

而且此方法还多了一个栈映射表，是JDK1.6 之后引入的一个JVM特性，主要用于提高 JVM 在加载类时对局部变量表中的类型验证效率

u1，u2，u4，分别代表无符号1个字节，2个字节和4个字节

constant_pool_count

常量池数量，值为下一个属性constant_pool的数量+1

constant_pool[constant_pool_count-1];

constant_pool是一个数组，他是一个cp_info类型，表示引用的各种字符串常量、类和接口名称、字段名称和其他常量

他根据tag的值分为不同的类型

cp_info类型的info属性不固定，根据不同tag分为不同的属性

比如：

CONSTANT_Methodref拥有class_index（方法所属类的索引）和name_and_type_index（方法的名称和描述符）

而CONSTANT_Class_info拥有name_index（类名索引）

而这个索引其实指向的是constant_pool数组的某某项

比如，我们随便挑一个

这个是个CONSTANT_Class_info类型，而他的属性name_index的值为164，也就是常量池数组的164项，constant_pool[163]，因为数组从0开始

他指向的是CONSTANT_Utf8类型，值存在bytes属性中，也就是StudyClassInterface字符串

不同类型对应的cp_info结构体不同，具体参照https://docs.oracle.com/javase/specs/jvms/se15/html/jvms-4.html#jvms-4.4

this_class

他的值指向常量池数组当中的CONSTANT_Class_info类型，代表了这个class文件代表的类或接口

super_class

super_class代表父类，值不可能为0，值为0说明是Object类，值代表常量池数组的索引-1

interfaces_count

代表实现的接口数

interfaces[interfaces_count]

代表实现的接口，占用两个字节，索引指向的是CONSTANT_Class_info类型

fields_count

字段数量

fields[fields_count]

然后又是一个field_info结构体，该这个fields数组仅包括由此类或接口声明的那些字段，它不包括从父类或父类实现的接口继承的字段。

JVM字节码分析

这里以execCmd方法为例

可以看到从字节码分析，StringBuffer底层用的是StringBuilder处理，循环跳转用goto语句，本地变量表存储着每一个变量，存储对应变量前置为null（占位符），本地变量表搭配栈完成运算，相比原生的机器指令还是比较容易看的，封装了多层的汇编指令，看懂JVM指令也是ASM操作的重要基础。
